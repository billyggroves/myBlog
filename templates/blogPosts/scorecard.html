ScoreCard

Tracks scores for various card games!

Posted by Bill Groves on April 8, 2018

ScoreCard is what I would consider to be my first real application. I was just coming out of Ohio University where I had just finished building my team's senior project for our capstone Management Information System (MIS) course. A C# .NET based, private social media web application with gamification in the form of badges that would allow employees to post, share, like, or even vote on their accomplishments. Building this project is where I discovered my passion of software development. After I finished my capstone project, I couldn't wait to start my next project. I tried to think of a project that would provide value in my life. At the time, my Girlfriend and I were always playing games in the evenings, especially card games and wasted so much paper writing our scores down. This gave me the inspiration to create ScoreCard to track scores and making a healthy impact on the environment while I'm at it.

Requirements:

I tried to utilize as many resources from my course work as possible. As an MIS major, the actual process of software development is stressed more than the actual code. So, I started off with some requirements gathering.

To start off I tried to ask myself what I wanted from this application while remaining as unbiased as possible. I knew that there would be potential for there to be other users as well. So, I also asked others what they would want from this type of application. I then took all of this collected data and wrote them out as user stories in cards in a Trello board. Now, that I had some sort of idea of what I wanted to build it was time to figure out how to build it.

Since, the only code I had been exposed to prior to building ScoreCard, was C# .NET through my MIS courses, I first considered just building it as a web application. However, I really needed it to be mobile-based. I then made the decision to build it as an android application. I chose android because I have always had android phones and was a big fan of the android development podcast, Fragmented. This gave me the motivation to learn new tools and skills to build ScoreCard as an android application. I utilized sources, such as video courses on udemy.com and Lynda.com to learn the basic concepts of Java as well as android itself.

Structure:

I developed this application with XML for the frontend, Java with Android Framework for backend, Realm ORM for Java(Android) for the database, and Git and GitHub for version control. I chose to keep the app just standard vanilla Java Android with XML, because that is what I had been using in tutorials and contained the most accessible sources to learn from. However, now I would be very interested in remaking ScoreCard in Kotlin. I chose to use Realm to store the data, because I had used Android’s SQLite in some of the tutorials I had completed, but I wanted to try something new. Since, I was already comfortable with SQL in general, having taken multiple courses devoted to databases and data manipulation with SQL, I just wanted to look for something that was a little different. That’s when I heard Donn Felker one of the hosts from Fragmented talk about Realm. So, I figured I would give it a try. This was my first time ever using an ORM and learning an API without a tutorial and solely from the documentation and some example code.

This was my first self-taught and self-motivated project and it allowed me to prove to myself that I can create/learn anything I want. You can view the code on my GitHub account, go to https://github.com/billyggroves/ScoreCard.
