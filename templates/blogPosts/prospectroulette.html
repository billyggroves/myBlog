Prospect Roulette

My prospect tracking application!

Posted by Bill Groves on April 6, 2018

Prospect Roulette is a web application I made while working as a sales representative to track my prospects. We were provided with an ERP program that included a simple CRM web program for the sales team. This CRM system was very frustrating and provided me with very little help. In fact, the process of adding new prospects and entering notes for the prospects are on two completely different programs. I would have to enter the prospects information using a remote desktop. Then, I would have to log into the web application that was connected to the same database to enter notes for those prospects. Once, you log in you must search through the entire database of prospects for your branch office to find the one you just entered. I wanted to make something that was simple and quick to use. This is where I came up with the idea for Prospect Roulette.

Development Process:

I started off by gathering requirements for Prospect Roulette. I knew I wanted to quickly be able to look at a list of my prospects or my clients. So, I created two HTML pages, one for each. I knew that I wanted to be able to quickly insert clients or prospects into the application. I created an HTML page for inserting data. While, determining the requirements, I could not think of what I would want the homepage to look like. I thought of an idea, where I would write an algorithm which would pull ten prospects that I hadn't contacted for at least a week. This would be a great reminder for some of the prospects which I may have forgotten about (This was the inspiration for me to create the name for Prospect Roulette). So, I created an HTML page for the homepage.

Checks if the user has not contacted the random prospect within a week. If they have, the current iteration of the loop is reset:

I chose to build this application using bootstrap, python with the Flask framework, jQuery, and SQLAlchemy. These selections were largely influenced by the edx.org course, CS50. This course is where I learned how to program in C, python, Flask framework and gave me a more in-depth understanding of general computer science concepts. I chose these, because I was most familiar with these languages and frameworks and felt that I would be able to create this project very efficiently in terms of time and quality. However, time was the most important since I really needed something as quick as possible, to help me save time and produce more sales.

The backend contains the python (Flask) code which create the lists, generate the data onto the HTML pages, and insert the data into the database. With all the data transferring onto the HTML pages, I then created a more interactive UI/UX with jQuery. This enabled the user to click on a company’s <div> then that <div> would expand and show a list of all the notes for the company as well as all the company’s contact information. Then, the user was able to add notes and contact information straight from the list. I also made the data be transferred through jQuery with ajax to keep the page from refreshing, so the user could insert data more quickly. This made for a much cleaner experience for the user.

Finally, I deployed this application onto Heroku with a PostgreSQL database. This was the biggest learning curve for me, because it was the first time I had deployed any project. Since, all the programs that I had created before, were all just local projects that I did not need to deploy. I chose to deploy Prospect Roulette. I chose to host my application on Heroku, because it was free for the most part, however, it was also easy to learn and integrates well with the Flask framework.

For a more indepth look at my code please go to https://github.com/billyggroves/prospectRoulette.
