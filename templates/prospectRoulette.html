Prospect Roulette

My prospect tracking application!

Posted by Bill Groves on March 1st, 2018

Prospect Roulette is a web application that I made while working as a sales representative in order to track my prospects. We were provided with ERP software that included a very simple prospect tracker. However, the system was very frustrating and provided me with very little help. In fact, the process of adding new prospects and entering notes for the prospects are on two completely different interfaces. So, I would have to enter the prospects using a remote desktop and then use the web application that was connected to the same database to enter notes for those prospects. Then, once you try to find the prospects that you have entered, you have to filter through all the prospects and clients for your whole branch. So, this drove me crazy and wanted to make something that was simple and quick to use. This is where I came up with the idea for Prospect Roulette.

Development Process:

I chose to build this application using bootstrap, python with the Flask framework, jQuery, and SQLAlchemy. These selections were largely influenced by the edx.org course, CS50. Where I learned how to program in C, python, Flask framework, and gave me a more in-depth understanding of general computer science concepts. I chose these, because I was most familiar with these languages and frameworks and felt that I would be able to create this project very efficiently in terms of time and quality. However, time was the most important since I really needed something as quick as possible, to help me save time in order to produce more sales.

I started off by performing quick requirements for Prospect Roulette. I knew I wanted to quickly be able to look at a list of my prospects or my clients. So, I created two HTML pages, one for each. I knew that I wanted to be able to quickly insert clients or prospects into the application. So, I created an HTML page for inserting data. While, determining the requirements, I could not think of what I would want the homepage to look like. I finally thought of a good idea, where I would write an algorithm which would pull ten prospects that I hadn't contacted for at least a week, that way I would be able to get reminded of some that I may have forgotten about (This was the inspiration for me to create the name for Prospect Roulette). So, I created an HTML page for the homepage.
I then created the backend where I had the python(Flask) code create the lists, generate the data onto the HTML pages, and insert the data into the database. Once, I had all the data forming onto the HTML pages, I then created a more interactive UI/UX with jQuery. I made it where if the user would click one of the prospects or clients, then that div would expand and show a list of all the notes for the system as well as all the contact information. Then, the user was able to notes and contact information straight from the list. I also made it, so the data was passed through jQuery with ajax in order to keep the page from refreshing, so the user could insert data more quickly.
Finally, I deployed this application onto Heroku with a create PostgreSQL database. This was the biggest learning curve for me, because it was the first time I had actually deployed any project. Since, all of the programs that I had created before, were all just local projects that I did not need to deploy. I chose to deploy Prospect Roulette. I chose to host my application on Heroku, because it was free for the most part, however, it was also easy to learn and works very well with the Flask framework.
